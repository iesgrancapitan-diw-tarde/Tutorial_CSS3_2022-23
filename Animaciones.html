TUTORIAL CSS3 - Animaciones

Las animaciones nos permiten animar la transición entre un estilo CSS y otro

animation-delay

Indica el tiempo de retardo que existe entre el momento que el elemento se carga y el comienzo de la secuencia de animación

Sintaxis

animation-delay: [tiempo en segundos(s) o milisegundos (ms)];
animation-delay: [tiempo en segundos(s), tiempo en milisegundos(ms)];
animation-delay: [valores globales (inherit, revert...)];

Admite uno o dos valores, que son los siguientes:

  • Un valor: Se aplica al espacio de tiempo que debe transcurrir desde que la animación se aplica al elemento y el comienzo de la animación
  • Dos valores: Cuando se especifican varios valores separados por comas en una propiedad animación, se aplican las animaciones en el orden en que aparecen.
  • Valor Negativo: Un valor negativo hace que la animación comience inmediatamente, pero a mitad de su ciclo. Por ejemplo, si especifica -1s como tiempo de retraso de la animación, la animación comenzará inmediatamente, pero comenzará 1 segundo en la secuencia de animación.

Ejemplos:

animation-delay: 2s;
animation-name: rotate;
animation-duration: 2s

Resultado:

animation-duration

Indica el tiempo que tarda una animación en completar un ciclo

Sintaxis

animation-duration: [tiempo en segundos(s) o milisegundos (ms)];
animation-duration: [tiempo en segundos(s), tiempo en milisegundos(ms)];
animation-duration: [inherit | initial | revert | revert-layer | unset];

Admite uno o dos valores (siempre positivos), que son los siguientes:

  • Un valor: Se aplica al espacio de tiempo que debe transcurrir desde que la animación se aplica al elemento y el comienzo de la animación
  • Dos valores: Cuando se especifican varios valores separados por comas en una propiedad animation-*, se aplican a las animaciones en el orden en que aparecen los nombres de animación. Para situaciones en las que el número de animaciones y los valores de propiedad de animación-* no coinciden, consulte Configuración de múltiples valores de propiedad de animación.

Ejemplos:

animation-name: scale;
animation-duration: 2s

Resultado:

animation-name

Indica los nombres de una o más reglas de @keyframes que describen la animación que se aplicará a un elemento

Sintaxis

animation-name: animacion1;
animation-name: animacion1, animacion2;
animation-name: [inherit | initial | revert | revert-layer | unset];

Admite uno o dos valores (siempre positivos), que son los siguientes:

  • none: No hay animación que se aplique al elemento. Esto es equivalente a no especificar la propiedad animation-name. Se puede utilizar para desactivar una animación.
  • Personalizado:Es un identificador que se compone de una combinación de letras que distinguen entre mayúsculas y minúsculas de a-Z y dígitos de 0-9, incluidos los guiones medios (-) y guiones bajos (_), pero no puede comenzar con un guión medio o guión bajo.

Ejemplos:

  • Ejemplo 1:
    • 
.boxNameTranslate:hover {
    animation-name: translate;
    animation-duration: 2s;
}
                
    // CSS de @keyframe "animation-name"
@keyframes translate {
    0% {
        transform: translate(0);
    }
    100% {
        transform: translate(100px);
    }
}
  • Ejemplo 2:
    • 
.boxNameSkew:hover {
    animation-name: skew;
    animation-duration: 2s;
    animation-direction: normal;
}

    // CSS de @keyframe "animation-name 2"
@keyframes skew {
    0% {
        transform: skew(0);
    }
    100% {
        transform: skew(45deg);
    }
}

animation-direction

Indica si una animación debe reproducirse hacia adelante, hacia atrás o alternar hacia adelante y hacia atrás entre la reproducción de la secuencia

Sintaxis

animation-direction: [normal | reverse | alternate | alternate-reverse];
animation-direction: dirección1, dirección2;
animation-direction: [inherit | initial | revert | revert-layer | unset];

Admite cuatro valores, que son los siguientes:

  • normal: La animación se reproduce hacia adelante cada ciclo. En otras palabras, cada vez que la animación se repita, la animación se restablecerá al estado inicial y comenzará de nuevo. Este es el valor predeterminado.
  • reverse:La animación se reproduce al revés cada ciclo. Es decir, cada vez que la animación se repita, la animación se restablecerá al estado final y comenzará de nuevo. Los pasos de animación se realizan hacia atrás, y las funciones de temporización también se invierten.
  • alternate:La animación invierte la dirección de cada ciclo, siendo la primera iteración hacia adelante. El recuento para determinar si un ciclo es par o impar comienza en uno.
  • alternate-reverse:La animación invierte la dirección de cada ciclo, y la primera iteración se reproduce al revés.

Ejemplos:

  • Ejemplo 1:
    • 
.boxDirectionNormal:hover {
    animation-name: translateNormal;
    animation-duration: 2s;
    animation-direction: normal;
    animation-iteration-count: infinite;
}
  • Ejemplo 2:
    • 
.boxDirectionAlternate:hover {
  animation-name: translateAlternate;
  animation-duration: 2s;
  animation-direction: alternate;
  animation-iteration-count: 4.2;
}

    (Los ejemplos vistos son similares a reverse y alternate-reverse en sentido contrario)

animation-iteration-count

Indica el número de veces que se debe reproducir una secuencia de animación antes de detenerse.

Sintaxis

animation-iteration-count: [infinite | number];
animation-iteration-count: iteración1, iteración2;
animation-iteration-count: [inherit | initial | revert | revert-layer | unset];

Admite cuatro valores, que son los siguientes:

  • infinite: La animación se reproduce para siempre
  • number:El número de veces que se repetirá la animación; esto es 1 por defecto. Puede especificar valores no enteros para formar parte de un ciclo de animación: por ejemplo, 0.5 reproducirá la mitad del ciclo de animación. Los valores negativos no son válidos.

Ejemplos:

  • Ejemplo 1:
    • 
.boxIterationNumber:hover {
  animation-name: rotate3d;
  animation-duration: 2s;
  animation-direction: alternate;
  animation-iteration-count: 2;
}

    
@keyframes rotate3d {
  0% {
    transform: perspective(400px) rotate3d(1, 1, 1, 0deg);
  }
  100% {
    transform: perspective(400px) rotate3d(1, 1, 1, 360deg);
  }
}
  • Ejemplo 2:
    • 
.boxIterationInfinite:hover {
  animation-name: rotate3d;
  animation-duration: 0.5s;
  animation-direction: alternate;
  animation-iteration-count: infinite;
}

animation-play-state

Indica si una animación se está ejecutando o en pausa.

Sintaxis

animation-play-state: [paused | running];
animation-play-state: estado1, estado2;
animation-play-state: [inherit | initial | revert | revert-layer | unset];

Admite cuatro valores, que son los siguientes:

  • runninf: La animación se activa
  • paused:la animación se pausa

Ejemplos:

  • Ejemplo 1:
    • .boxPlayState {
  ...
  animation-play-state: paused;
}

.boxPlayState:hover {
  animation-play-state: running;
}

    
@keyframes rotateSquare {
  0% {
    transform: rotate(0);
  }
  100% {
    transform: rotate(360deg);
  }
}

animation-timing-function

Indica cómo progresa una animación a lo largo de la duración de cada ciclo.

Sintaxis

animation-timing-function: [ease | ease-in | ease-out | ease-in-out | linear | step-start | step-end];
animation-timing-function: cubic-bezier(num1, num2, num3, num4), steps(integer, step-position);
animation-timing-function: [inherit | initial | revert | revert-layer | unset];

Admite cuatro valores, que son los siguientes:

  • ease: Igual a cubic-bezier (0,25, 0,1, 0,25, 1,0), aumenta en velocidad hacia la mitad de la animación, disminuyendo al final.
  • ease-in: Igual a cubic-bezier(0.42, 0, 1.0, 1.0), comienza lentamente, con la velocidad de transición de la propiedad de animación aumentando hasta que se completa.
  • ease-out: Igual a cubic-bezier(0, 0, 0.58, 1.0), comienza rápidamente, ralentizando la animación continúa.
  • ease-in-out: Igual a cubic-bezier(0,42, 0, 0.58, 1.0), acelerando y luego disminuyendo de nuevo.
  • linear: Igual a cubic-bezier (0,0, 0,0, 1,0, 1,0), se anima a una velocidad uniforme.
  • cubic-bezier: Define una curva de Bézier personalizada. Los valores numéricos deben estar entre 0 y 1.
  • steps: Define una función de paso personalizada. El primer valor numérico define el número de pasos, y el segundo valor define el punto de inicio de cada paso.
  • step-start: Igual a steps(1, start), la animación se divide en pasos, con cada paso comenzando en el momento en que se completa el paso anterior.
  • step-end: Igual a steps(1, end), la animación se divide en pasos, con cada paso comenzando en el momento en que se completa el paso anterior.

Ejemplos:

  • Ejemplo 1:
    • .fillForwards:hover {
  animation-name: scaleTiming;
  animation-duration: 0.5s;
  animation-fill-mode: forwards;

  animation-iteration-count: 4;
}

    
@keyframes scaleTiming {
  0% {
    transform: scale(1);
  }
  100% {
    transform: scale(2);
  }
  • Ejemplo 2:
    • .fillBackwards:hover {
  animation-name: scaleTiming;
  animation-duration: 0.5s;
  animation-fill-mode: backwards;
  animation-iteration-count: 4;
}

    
@keyframes scaleTiming {
  0% {
    transform: scale(1);
  }
  100% {
    transform: scale(2);
  }

animation-fill-mode

Indica cómo una animación CSS aplica estilos a su objetivo antes y después de su ejecución.

Sintaxis

animation-fill-mode: [none | forwards | backwards | both];
animation-fill-mode: estado1, estado2;
animation-fill-mode: [inherit | initial | revert | revert-layer | unset];

Admite cuatro valores, que son los siguientes:

  • none: La animación no aplicará ningún estilo, en su caso utilizará cualquier otra regla CSS aplicada a él. Este es el valor predeterminado.
  • forwards: Se conservarán los valores establecidos por el último fotograma clave encontrado durante la ejecución. El último fotograma clave depende del valor de la animación-dirección y el recuento de animación-iteración
  • backwards: Se conservarán los valores establecidos por el primer fotograma clave encontrado durante la ejecución. El último fotograma clave depende del valor de la animación-dirección y el recuento de animación-iteración:
  • both: La animación seguirá las reglas tanto hacia adelante como hacia atrás, extendiendo así las propiedades de la animación en ambas direcciones.

Ejemplos:

  • Ejemplo 1:
    • .fillForwards:hover {
  animation-name: scaleTiming;
  animation-duration: 0.5s;
  animation-fill-mode: forwards;

  animation-iteration-count: 4;
}

    
@keyframes scaleTiming {
  0% {
    transform: scale(1);
  }
  100% {
    transform: scale(2);
  }
  • Ejemplo 2:
    • .fillBackwards:hover {
  animation-name: scaleTiming;
  animation-duration: 0.5s;
  animation-fill-mode: backwards;
  animation-iteration-count: 4;
}

    
@keyframes scaleTiming {
  0% {
    transform: scale(1);
  }
  100% {
    transform: scale(2);
  }
